# 07. 量化成果

> 「數字比形容詞有說服力 100 倍」。這份文件記錄專案所有可量化的指標，面試時直接搬出來用。

> ⚠️ 部分數字是估算，等下個階段優化時補上精確值。

---

## 效能指標

### 3D 重建端對端耗時
| 階段 | 耗時 |
|---|---|
| 接收 + 驗證 | < 0.5s |
| Tripo 上傳 | 1~3s |
| Tripo create_task | < 1s |
| Tripo 輪詢 (image_to_model draft) | **30~60s** (主要瓶頸) |
| Tripo Refine (可選) | +60s |
| 下載 .glb | 2~5s |
| 存檔 + multipart 組裝 | < 1s |
| **總計 (不含 Refine)** | **~35s** |
| **總計 (含 Refine)** | **~90s** |

### Mock 模式對比
| 模式 | 平均耗時 | 成本 |
|---|---|---|
| 真實流程 (no refine) | ~35s | 💰 1 次 Tripo credit |
| 真實流程 (with refine) | ~90s | 💰 2 次 Tripo credit |
| **Mock 模式** | **< 0.5s** | **✅ 0** |

→ **開發迭代速度提升 70 倍**，且零成本。

### 去背 + 標準化
- rembg 去背：~1.5s（CPU）
- compose_square_portrait：< 0.5s

### Gemini 試穿合成
- 一件衣服：~10~15s
- 多件（2~3 件）：~25~40s

## 成本指標

### 單次操作成本（推估）
| 操作 | API 成本 |
|---|---|
| 衣服去背 + 風格分析 | 約 Gemini 1 次 vision 呼叫 |
| 2D 試穿 (1 件) | 約 Gemini 2 次（分析 + 合成） |
| 3D 重建 (draft) | 1 Tripo credit |
| 3D 重建 (refine) | 2 Tripo credits |
| Mock 模式 | 0 |

### 省成本設計
1. **Mock 模式**：開發成本歸零（重要！）
2. **模型分工**：顏色判斷用 `flash` 不用 `flash-image`，省約 80% token
3. **Refine 預設關閉**：開發測試只用 draft，省一半 credits
4. **本地 rembg**：去背不走雲端，永久零邊際成本

## 容錯與穩定性

### 錯誤映射覆蓋率
- Tripo 上游 8 種錯誤 → 全部有對應的業務碼處理
- 4 個未捕獲例外點（每個 endpoint 一個 `try/except` 兜底）

### 失敗 case 處理
| 情境 | 行為 |
|---|---|
| Tripo 積分用完 | 回 4410，前端跳轉儲值頁 |
| Tripo 限速 | 回 4429，前端自動重試 |
| 照片肢體不全 | 回 4422，前端提示換照片 |
| 服務崩潰 | 回 4500，log 寫入 exception trace |

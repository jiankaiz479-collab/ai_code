# 3D 模型重建 (Tripo) 的輸入影像解析度

目前送進 3D 重建模組 (Tripo3D) 的影像解析度是 **1024 × 1024**，而不是 512 × 512。

## 詳細說明

根據系統的「人像標準化」模組（`/fitting/modules`）設計，影像進入 3D 重建流程前，後端會強制進行以下預處理：
1. **去背**：去除複雜背景，只保留主體。
2. **置中對齊**：將人物主體放置於畫面中央。
3. **標準化裁切與縮放**：統一處理成 **1024 × 1024** 像素，並設定 5% 的邊緣留白。

這樣的標準化規格 (Hardcode) 能夠確保 Tripo3D 接收到的輸入圖片擁有最高的一致性，比依賴 AI 自行裁切還要穩定 100 倍。

## 2D 去背與試穿的影像預處理規格

相較於 3D 強制的正方形規格，進入 2D 去背（rembg）或 2D 試穿合成流程的圖片，為了保留原始細節與真實長寬比，會套用以下的「強健預處理 (Robust Preprocessing)」規格：

1. **解析度上限 (Resize)**：限制長邊不超過 **2048 像素**，並進行等比縮放（避免如 iPhone 12MP 原圖導致去背耗時過久、記憶體吃滿及邊緣抖動）。
2. **色彩空間 (Color Profile)**：強制轉換為 **sRGB** 色彩空間（解決手機 P3 廣色域造成的「紅衣變橘色」等色偏問題）。
3. **方向校正 (EXIF Rotation)**：強制讀取並套用 EXIF 旋轉標籤，確保直拍的照片在 AI 處理時不會變成橫躺。
4. **格式正規化**：將 HEIC / WEBP / PNG / JPG 等多種格式，在進入 AI 前統一正規化為標準的 RGB 陣列。

## 影像輸入規格 (依模組分類)

### 1. 3D 模型重建 (Tripo)
- **核心目標**：追求極致的「一致性」與「穩定性」
- **解析度 / 尺寸限制**：強制縮放為 **1024 × 1024**
- **長寬比**：強制 **1:1 (正方形)**
- **色彩與方向校正**：依賴前置預處理
- **特殊影像處理**：強制去背、主體置中、邊緣 **5% 留白**

### 2. 2D 試穿合成 (Gemini)
- **核心目標**：保留「原始細節」與「真實長寬比」
- **解析度 / 尺寸限制**：限制長邊不超過 **2048 像素**（等比縮放）
- **長寬比**：**不限制**（保留原始比例）
- **色彩與方向校正**：強制轉換 **sRGB**、套用 **EXIF 旋轉**
- **特殊影像處理**：格式正規化（統一轉標準 RGB 陣列）

### 3. 去背 (rembg)
- **核心目標**：追求「主體精準分割」與「邊緣平滑」
- **解析度 / 尺寸限制**：限制長邊不超過 **2048 像素**（等比縮放）
- **長寬比**：**不限制**（保留原始比例）
- **色彩與方向校正**：強制轉換 **sRGB**、套用 **EXIF 旋轉**
- **特殊影像處理**：Alpha Matting、連通域過濾、格式正規化
